# 🏟️ vieWhere - 진행 계획 및 태스크

이 문서는 프로젝트의 전체 계획, 진행 상황, 그리고 각 단계별 검증(Verification) 항목을 눈으로 확인하고 트래킹하기 위한 체크리스트입니다.

## 🛠️ 단계별 체크리스트

### Phase 1: 프로젝트 기반 설정 및 디자인 시스템 구축
- [x] `vieWhere` 프로젝트 폴더 구성 및 관리용 초기 기획 문서 작성
- [x] Vite 템플릿을 활용한 React + TypeScript 환경 초기화 (`npx create-vite`)
- [x] 프로젝트 필요 패키지 설치 및 쓰이지 않는 더미 템플릿 파일 정리
- [x] 프리미엄 UI 디자인 구상을 위한 **바닐라 CSS 디자인 토큰** 구성 (컬러 팔레트 변수, 폰트, 애니메이션 계수 등)
- [x] 홈(Home) 및 뷰 사이드의 글로벌 레이아웃 컴포넌트 뼈대 작성
- **✅ Phase 1 검증 항목:**
  - [x] 로컬 실행(`npm run dev`) 시 에러 없이 기본 레이아웃이 브라우저에 표시되는가?
  - [x] 모바일 크기부터 와이드 모니터까지 창 너비를 조절했을 때 레이아웃 뼈대가 무너지지 않는가?

### Phase 2: 가상 야구장 맵 설계 및 액션 테스트 (PoC)
- [x] 필수 구역(1루, 3루, 센터, 외야 등)을 모사한 심플한 가상의 야구장 SVG 구조 모델링 (`VirtualStadiumMap.tsx`)
- [x] 우측(혹은 하단)에 구역명과 시야 이미지를 출력하는 UI 컴포넌트 생성 (`SectorViewPanel.tsx`)
- [x] 가상 구역 데이터 파일 생성(JSON 형태) 및 SVG ID 요소와 상호 연결
- [x] 각 SVG 영역(구역)에 마우스 Hover 및 클릭 시 활성화되는 CSS 애니메이션 추가
- [x] 클릭 이벤트를 State로 관리하여 패널 컴포넌트에 즉시 연동 반영
- **✅ Phase 2 검증 항목:**
  - [x] 마우스 호버 시 지정된 가상 SVG의 구역만 깔끔하게 시각적 강조 효과가 나타나는가?
  - [x] 특정 구역 클릭 시 사진/정보 패널에 해당 구역의 텍스트와 임시 이미지가 정상적으로 출력되는가?

### Phase 2.5: 형상 관리 및 GitHub 레포지토리 연동
- [x] 로컬 git 초기화 (`git init`)
- [x] Phase 1, 2에서 완성 및 검증된 가상 구장 테스트 베드 코드를 초기 커밋 달성
- [x] 원격 GitHub 레포지토리 생성 및 연결 (`git remote add origin ...`)
- [x] 코드 푸시(Push) 및 브랜치 전략 설정(향후 SSG 랜더스 데이터를 위한 브랜치 대비)

### Phase 3: SSG 랜더스필드 본 데이터 확장 돌입
- [ ] SSG 랜더스필드의 실제 좌석 구역 형태(응원지정석, 프렌들리존, 일반석 등)를 모사한 전용 SVG 설계 (`SSGLandersMap.tsx`)
- [ ] 각 구역별 실제 뷰와 대응하는 사진 이미지 수급(혹은 테스트용 실제 구장 이미지 URL 배치) 및 정보 매핑
- [ ] 더미 JSON 데이터를 실제 SSG 랜더스필드 규격에 맞는 데이터베이스 구조로 전면 교체 적용
- [ ] 가상 야구장을 본 야구장 모델로 렌더링 스왑다운 및 기능 통합
- **✅ Phase 3 검증 항목:**
  - [ ] SSG 랜더스필드의 복잡한 구역 경계선들이 찌그러지거나 탈선 없이 깔끔하게 모바일에서 출력되는가?
  - [ ] 디스플레이 화면의 매핑이 누락되는 구간 없이 작동하는가?

### Phase 4: 폴리싱 및 사용자 편의성 최적화
- [ ] UI 요소 전반의 부드러운 Glassmorphism, 마이크로 애니메이션 트랜지션 고도화
- [ ] 처음 접속한 사용자를 위한 간단한 '툴팁' 가이드(예: 구역을 터치/클릭해보세요) 적용

- **✅ Phase 4 검증 항목:**

  - [ ] 스마트폰 터치/스와이프 시 뷰어 구역 클릭 작동이 자연스럽게 유지되는가?
