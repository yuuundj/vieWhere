# 📈 vieWhere - 초기 기획 및 로드맵

이 문서는 프로젝트가 시작될 때 합의된 초기 청사진과 기획 방향성을 보존하기 위한 문서입니다.

## 🎯 핵심 기획 목표
프로젝트의 최우선 목적은 **사용자가 야구장 좌석 배치도의 특정 구역을 선택했을 때, 해당 구역에서의 생생한 야구장 뷰(시야)를 모바일 및 직관적인 웹 환경에서 즉시 확인**할 수 있도록 하는 것입니다. 이를 우선적으로 구현하기 위해 다른 부가적인 기능들은 배제하고 인터랙션에 집중합니다.

## 💡 수립된 핵심 방향성 (피드백 반영 결과)
1. **단위 세분화 (개별 좌석 ➡️ 구역(Sector) 단위)**: 
   너무 세분화된 개별 좌석 대신, 실질적인 시야 차이를 경험할 수 있는 '구역 또는 블록(예: 1루 응원지정석 101블록)' 단위로 뷰를 제공하는 것으로 스코프를 최적화했습니다.
2. **가상 구장을 통한 기능 동작 최우선 검증 (PoC)**: 
   처음부터 디테일한 실제 야구장 데이터를 구축하다가 로직이 엉키는 것을 막고자, 단순한 형태의 가상 야구장 SVG 구조와 테스트 이미지를 기반으로 뼈대가 문제없이 동작하는지 먼저 테스트합니다.
3. **기록 도입 시점 (GitHub 연결)**:
   가상 야구장 맵에서의 인터랙션 및 화면 렌더링 테스트가 성공적으로 끝나면, 즉각 로컬 코드를 GitHub 레포지토리에 연동합니다. 이때부터 실 서버 배포를 염두에 두고 진행 상황을 커밋(Commit)해 나갑니다.
4. **첫 번째 실적용 타겟 (SSG 랜더스필드)**: 
   GitHub 연결이 완료된 후, 코어 기능을 토대로 실제 SSG 랜더스필드의 SVG 맵 파츠와 구역 실데이터를 교체 투입하여 본격적인 서비스 형태를 빌드합니다.
